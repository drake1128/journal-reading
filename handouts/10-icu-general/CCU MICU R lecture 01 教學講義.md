## 機械通氣管理


### 呼吸器基本操作


criticalbox
進ICU第一天一定會非常緊張，先弄清楚主治醫師希望你做什麼
criticalbox


#### 學習策略


- 看到呼吸治療師來了，就跟上去學習
- 醫院的呼吸治療師訓練很好，可以直接請教
- 弄清楚病人上呼吸器的原因
- 了解今天的X光、今天的血氧、今天的ABG怎麼樣


#### 基本操作技能


- 學會按resistance和compliance
- 了解機器的alarm系統
- 按掉alarm之前記得看一下在alarm什麼
- RT可以教你大概10分鐘就學會基本操作


### Ventilator Weaning評估


tipbox
**Weaning重點：**頭3天的重點是評估病人原發疾病是否真的改善
tipbox


#### Weaning評估標準


- 原發疾病改善
- 血行動力學穩定
- 意識清楚，能配合
- 痰量適中
- RSBI < 105 breaths/min/L


#### 插管指標


- 當護理師反覆提醒要不要插管時，要認真看待
- 有疑惑時應該問上級，而不是等到CPR再插管
- 護理師一定會比你還急


## 機械通氣急性惡化的處置


### Acute Deterioration Under Mechanical Ventilation


criticalbox
**系統性處置流程**
criticalbox

當病人在呼吸器治療下發生急性惡化時，需要系統性的評估和處置：


| |p{5cm}|p{7cm}|} 呼吸器急性惡化處置流程 |  |  |
| --- | --- | --- |
| **步驟** | **評估項目** | **處置方法** |
| 1. 立即評估 | 生命徵象、血氧、血壓、心率 | 準備急救藥物和設備 |
| 2. 機器檢查 | 呼吸器功能、電源、O₂接頭 | **立即使用Ambu bag排除機器問題** |
| 3. 管路檢查 | ETT位置、是否阻塞、漏氣 | 檢查ETT深度、吸痰、更換ETT |
| 4. 病人評估 | 氣胸、Auto-PEEP、人機對抗 | 胸部X光、調整鎮靜、止痛 |
| 5. 特殊處置 | 疑似氣胸時的處理 | 如果血壓下降急劇，不等X光，必要時以手術刀與Kelly (鈍頭) 進行減壓 |


### 氣胸的緊急處置


#### 臨床情境

以下是常見的臨床情境：

- 68歲COPD病人，pneumonia插管後躁動
- 血氧掉到81
- 心跳快到103，右邊呼吸音很小
- MV量在調節中掉到只有105


#### 處置選項


warningbox
**決策考量：**

- 這個病人入院時理學檢查即知道: 病人右邊呼吸音本來就較小
- Echo評估可能不是很把握
- 血壓開始下降（180 → 150 → 120 mmHg, SBP）


warningbox

**處置步驟：**

1. **立即使用Ambu bag：**排除機器問題，評估人機配合
2. **給予止痛藥：**避免給到muscle relaxation
3. **升壓劑：**先穩定血壓
4. **評估氣胸：**X光、超音波、臨床判斷
5. **必要時減壓：**不要等X光，臨床決策優先


#### 安全的減壓技術


tipbox
**Kelly鑷子技術：**

- 用刀劃開皮膚，但不要用針
- 用Kelly鑷子（頭是圓的）撐開肋膜
- 看有沒有空氣跑出來
- 避免iatrogenic pneumothorax
- 位置：mid-axillary line第二肋間


tipbox


## 心血管急症處置


### 心因性休克


如果病人都知道是心因性休克：

- 不要再灌一大堆水
- 藥物治療沒有效時考慮機械性支持
- IABP、ECMO、緊急導管介入
- 主治醫師會判斷


### 心律不整


criticalbox
**VT/VF處置：**

- VT/VF一定要會認
- 該電擊就直接電擊
- 護理師會比你還急，看得出來大家在急你就跟上


criticalbox

## 臨床溝通技巧


### SBAR溝通模式


protocolbox
**有效溝通的關鍵：**先設定好你打電話希望上級做什麼
protocolbox


#### 溝通原則


- 要很明確告訴上級：建議來看病人、建議做導管、需要幫忙評估
- 先把目的講清楚，再交代病人現況背景
- 某些心臟科醫師希望你反過來講


#### 實例

*"我有個昨天進來的病人76歲男性，來的時候胸痛，本來都不痛了，剛剛又痛起來，我看心電圖應該變成STEMI了，你可以來看嗎？"*

這樣大概15秒內就能讓上級了解狀況並立即行動。


## SBAR溝通技巧比較分析


### SBAR溝通模式在STEMI案例中的應用


criticalbox
**關鍵原則：**先明確告知你希望上級做什麼，然後15秒內讓上級了解狀況並立即行動
criticalbox


| |p{6.5cm}|p{6.5cm}|} SBAR溝通品質比較 - STEMI案例分析 |  |  |
| --- | --- | --- |
| **SBAR組成** | **糟糕的SBAR範例** | **優秀的SBAR範例** |
| **完整對話範例** |  |  |
| **完整對話** | ****糟糕範例：****     "老師你好，我是蔡醫師，我想要跟你報告一個病人的狀況。這個病人是76歲的男性，他有高血壓跟糖尿病的病史，平常有在服用降血壓藥物跟降血糖藥物。他昨天因為胸痛來急診，當時心電圖看起來好像沒有什麼特別的變化，所以就先收住院觀察。今天早上護理師說他又開始痛了，我去看了心電圖，好像跟昨天不太一樣，lead II, III, aVF有一些變化，看起來可能是下壁的心肌梗塞，不過我不是很確定，想請老師來看一下，你覺得呢？" | ****優秀範例：****     "老師，我是蔡醫師。我有個昨天進來的病人76歲男性，來的時候是胸痛，本來都不痛了，剛剛又痛起來，我看心電圖應該變成STEMI了，你可以來看嗎？" |
| **Situation (現況) 分析** |  |  |
| **Situation** | ****問題：****  - 開頭冗長，沒有緊急感 - 沒有立即說明急迫性 - 語氣不夠明確 | ****優點：****  - 立即說明基本資料 - 簡潔明確 - 突出急性變化："剛剛又痛起來" |
| **Background (背景) 分析** |  |  |
| **Background** | ****問題：****  - 過度詳細的病史 - 提及不相關的慢性病史 - 浪費寶貴時間 - 重點被稀釋 | ****優點：****  - 只提關鍵背景：昨天胸痛入院 - 強調病程變化：本來不痛了 - 避免不必要細節 |
| **Assessment (評估) 分析** |  |  |
| **Assessment** | ****問題：****  - 不確定的表達："好像"、"可能" - 缺乏信心 - 沒有明確判斷 - 描述過於技術性 | ****優點：****  - 明確判斷："應該變成STEMI了" - 展現臨床信心 - 直接了當 |
| **Recommendation (建議) 分析** |  |  |
| **Recommendation** | ****問題：****  - 沒有明確要求 - 把決定推給上級："你覺得呢？" - 沒有急迫性 - 缺乏具體建議 | ****優點：****  - 明確要求："你可以來看嗎？" - 暗示需要立即行動 - 簡潔有力 |
| **溝通效果比較** |  |  |
| **時間效率** | ****耗時過長：****  - 約60-90秒 - 上級需要重新整理資訊 - 可能需要追問 | ****高效簡潔：****  - 約15秒 - 上級立即了解狀況 - 可以立即行動 |
| **緊急感傳達** | ****缺乏急迫性：****  - 聽起來像一般會診 - 沒有time-sensitive的概念 - 可能延誤處置 | ****急迫性明確：****  - 傳達STEMI的急迫性 - 上級會立即反應 - 符合Door-to-balloon時間要求 |
| **上級反應** | ****可能的反應：**** "你慢慢說，到底現在是什麼狀況？心電圖變化在哪裡？" | ****期望的反應：**** "我立刻過去！同時通知心導管室準備！" |
| **學習重點分析** |  |  |
| **關鍵問題** | - 沒有預先設定目標 - 資訊過載 - 缺乏臨床自信 - 不了解急症特性 | - 目標明確 - 資訊精簡 - 展現判斷能力 - 符合急症處置原則 |
| **改善建議** | - 練習簡潔表達 - 建立臨床自信 - 了解time-sensitive diseases - 預先規劃溝通目標 | - 維持簡潔風格 - 持續建立臨床判斷 - 關注病程變化 - 保持溝通效率 |


### SBAR溝通的關鍵成功因素


| |p{11cm}|} SBAR溝通成功要素分析 |  |  |
| --- | --- | --- |
| **成功因素** | **具體表現** |  |
| **預先設定目標** | 打電話前就想清楚：我希望上級做什麼？來看病人？立即處置？還是電話指導？ |  |
| **15秒原則** | 在15秒內讓上級了解狀況的嚴重性和需要的行動 |  |

### 常見SBAR錯誤與避免方法


| |p{5cm}|p{6cm}|} SBAR常見錯誤分析 |  |  |
| --- | --- | --- |
| **常見錯誤** | **造成的問題** | **改善方法** |
| 沒有預先設定目標 | 溝通沒有重點，上級不知道要做什麼 | 打電話前先想：我要上級做什麼？ |
| 資訊過載 | 重點被稀釋，延誤決策 | 只提供關鍵資訊，避免無關細節 |
| 缺乏信心 | 大量使用"可能"、"好像" | 展現臨床判斷，承擔適當責任 |
| 沒有急迫性 | 延誤time-sensitive疾病處置 | 了解各種急症的時間要求 |
| 把決定推給上級 | "你覺得呢？"沒有建議 | 提出具體建議和要求 |
| 過度技術性描述 | 浪費時間，缺乏重點 | 用簡潔的臨床判斷取代技術描述 |

## 團隊融入與心態調整


### 成功的關鍵


tipbox
**融入團隊比表現優異更重要：**

- 不會要求第一天就表現非常棒
- 重點是融入團隊，跟護理師和NP合作
- 能明確觀察到護理站哪一床在變差
- 學習主治醫師和NP的思考模式


tipbox


### 角色定位


以第一天在加護病房的經驗為例：

- 學長希望住院醫師不要廢話太多
- 不要問太多問題
- 把早上8點到9點交班的事情在下午6點前都做完
- 學會講重點，讓學長用整個晚上確認你講的是真的


### 時間敏感性疾病的識別


criticalbox
**真正會讓人不高興的時間敏感疾病：**

- 一邊瞳孔放大，過了2個小時CT也沒做
- 這些都是早上9點就知道有變化的
- 要知道什麼事情是時間敏感的


criticalbox


## 學習資源與自我提升


### 現代學習方式


tipbox
**AI輔助學習：**

- 直接跟AI (ChatGPT, Claude,Grok) 說：我要進ICU了，對呼吸器調整很困惑很焦慮
- 詢問有什麼自學方案
- 10分鐘內就有答案
- 現在學習是完全不同的時代


tipbox


### 模擬演練


可以進行簡單的桌上演練：

- 5-7分鐘討論一個臨床情境
- 討論下一步選項有哪些
- 不用真的用假人，像打牌一樣討論即可
- 可以增加信心，知道怎麼做會最好
