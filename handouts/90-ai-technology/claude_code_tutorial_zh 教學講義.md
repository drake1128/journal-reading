## 簡介

本教學手冊提供 Claude Code 的完整使用指南。Claude Code 是 Anthropic 官方推出的 CLI 工具，專為 AI 輔助程式開發而設計。本手冊涵蓋安裝設定、基本操作、進階功能及最佳實踐，協助開發者充分發揮 Claude Code 的強大功能，大幅提升開發效率。

---

## 1. 簡介

Claude Code 是 Anthropic 開發的終端機介面 AI 程式助手。與其他支援多種模型的 AI 程式工具不同，Claude Code 專為 Claude 模型設計，實現工具與底層 AI 的深度整合與最佳化。

> **資訊**：Claude Code 直接與 Anthropic 的模型配合運作，省去中間環節，以具競爭力的價格提供最佳效能。

### 1.1 為什麼選擇 Claude Code？

Claude Code 的主要優勢包括：

- **大型檔案處理**：可靠地更新超過 18,000 行程式碼的檔案
- **複雜程式庫導覽**：擅長搜尋模式並理解各元件之間的關係
- **穩定性**：相較其他 AI 程式工具，極少發生卡住的情況
- **直接整合**：由開發底層模型的同一團隊打造
- **成本效益**：無中間商加價，直接享有 API 成本優惠

## 2. 安裝與設定

### 2.1 VS Code 擴充套件

Claude Code 官方擴充套件支援以下編輯器：

- Visual Studio Code
- Cursor
- Windsurf
- 其他 VS Code 分支版本

> **提示**：此擴充套件讓您能直接在 IDE 中啟動 Claude Code。您可以同時開啟多個 Claude Code 視窗，平行處理程式庫的不同部分。

### 2.2 自動載入檔案內容

當您在編輯器中開啟檔案時，Claude Code 會自動將其載入對話內容中。這種無縫整合減少了手動指定檔案的麻煩。

## 3. 終端機介面基礎

### 3.1 必備快捷鍵

| 快捷鍵 | 功能 |
|--------|------|
| Escape | 停止 Claude 目前的操作 |
| Escape（按兩次） | 檢視歷史訊息列表 |
| 上方向鍵 | 瀏覽過去的對話（包含先前的工作階段） |
| Shift + Enter | 在訊息中換行 |
| Ctrl + V | 從剪貼簿貼上圖片 |
| Shift + 拖曳 | 將檔案拖曳至終端機 |
| Ctrl + C（按兩次） | 退出 Claude Code（請小心使用！） |

> **警告**：請勿使用 Ctrl + C 停止 Claude 的工作——這會直接關閉程式。請改用 Escape 鍵。

### 3.2 檔案與圖片處理

**拖曳檔案**（在 VS Code/Cursor 中）：

1. 按住 Shift 鍵
2. 將檔案拖曳至 Claude Code 終端機
3. 檔案將被引用至您的對話中

**貼上圖片**：使用 Ctrl + V 從剪貼簿貼上圖片（標準的 Cmd + V 在 Mac 上無法貼上圖片）。

## 4. 模型選擇

### 4.1 可用模型

Claude Code 支援多種 Claude 模型：

- **Opus**：功能最強大的模型，建議用於複雜任務
- **Sonnet**：更具成本效益，適合一般任務

### 4.2 切換模型

使用 `/model` 指令切換模型：

```text
/model opus
/model sonnet
```

> **提示**：預設行為是使用 Opus 直到達到使用量上限的 50%，然後切換至 Sonnet。對大多數使用者而言，建議使用此自動切換機制。

### 4.3 模型比較

| 項目 | Opus | Sonnet |
|------|------|--------|
| 能力 | 最高 | 非常好 |
| 速度 | 快 | 快 |
| 成本 | 較高 | 更經濟 |
| 適用情境 | 複雜任務 | 一般任務 |

## 5. 必備斜線指令

### 5.1 /clear 指令

> **提示**：每次開始新任務時都要清除對話！您不需要讓舊的對話記錄消耗 token，避免壓縮（會額外執行 LLM 呼叫來摘要對話記錄）也能節省時間。

### 5.2 常用斜線指令

| 指令 | 說明 |
|------|------|
| `/clear` | 清除對話記錄 |
| `/model` | 切換 AI 模型 |
| `/help` | 顯示可用指令 |
| `/terminal-setup` | 設定終端機 |

## 6. 權限管理

### 6.1 權限問題

預設情況下，Claude Code 會詢問以下權限：

- 編輯檔案
- 執行 bash 指令
- 執行 lint/build 指令

當您離開後返回，發現 Claude 正在等待權限確認時，這可能會令人困擾。

### 6.2 跳過權限確認模式

若要略過權限提示：

```text
claude --dangerously-skip-permissions
```

> **警告**：此模式（類似 Cursor 的「YOLO 模式」）存在執行非預期指令的輕微風險。然而實際上，破壞性的 agent 行為極為罕見。請自行評估是否使用。

### 6.3 建議工作流程

1. 開啟 Claude Code 終端機
2. 按 Ctrl + C 停止預設啟動
3. 執行：`claude --dangerously-skip-permissions`
4. 開始不受中斷的工作

## 7. GitHub 整合

### 7.1 安裝 GitHub 應用程式

Claude Code 可與 GitHub 整合，自動審查 Pull Request：

```text
/install-github-app
```

### 7.2 設定程式碼審查

安裝後，Claude 會建立 `claude-code-review.yaml` 檔案。建議自訂：

```yaml
# claude-code-review.yaml
instructions: |
  尋找錯誤和安全問題。
  僅回報錯誤和潛在漏洞。
  保持簡潔。
```

> **提示**：AI 程式碼審查能發現人類可能遺漏的錯誤，因為它會對每一行程式碼投入一致的注意力，而人類可能會略過熟悉的模式。

### 7.3 AI 程式碼審查的優點

- 發現人類可能遺漏的真正錯誤
- 無論 PR 數量多寡，審查品質始終一致
- 識別安全漏洞
- 減少對程式風格的挑剔（正確設定後）

## 8. 訊息佇列

Claude Code 最強大的功能之一是訊息佇列。

### 8.1 佇列運作方式

1. 送出您的第一則訊息
2. 當 Claude 正在處理時，輸入更多訊息
3. 訊息會被排入佇列並依序處理
4. Claude 會智慧判斷何時處理佇列中的項目

### 8.2 工作流程範例

```text
您：為工具函式加入註解
[Claude 開始處理...]
您：也為主要的 JSX 區塊加入註解
[訊息已排入佇列]
您：以及為計算值加入註解
[訊息已排入佇列]
[Claude 依序處理所有任務]
```

> **提示**：Claude 對佇列的處理很聰明——如果需要您的回饋，它不會自動處理佇列中的訊息。請定期檢查是否需要您的輸入。

### 8.3 相較於傳統工作流程的優點

- 不需要另開記事本追蹤待辦任務
- Agent 不會閒置等待您的下一個提示
- 更有效率地利用時間——排入任務後稍後再回來查看

## 9. 自訂設定

### 9.1 CLAUDE.md 檔案

在專案根目錄建立 CLAUDE.md 檔案，為 Claude 提供：

- 專案概述
- 關鍵指令（build、lint、test）
- 專案特定的慣例

```markdown
# CLAUDE.md
## 專案概述
這是一個使用 TypeScript 的 React 應用程式。
## 關鍵指令
- 建置：`npm run build`
- 程式碼檢查：`npm run lint`
- 測試：`npm run test`
## 慣例
- 使用函式元件
- 使用 Tailwind CSS 進行樣式設計
```

> **提示**：CLAUDE.md 檔案可以是階層式的。在專案層級放置一個，並在巢狀目錄中放置額外的檔案。Claude 會在相關時優先使用最具體（最深層巢狀）的檔案。

### 9.2 自訂 Hooks

Hooks 讓您能在特定時間點自動執行指令：

- **編輯被接受前**：對特定檔案執行 Prettier
- **編輯後**：對修改的檔案執行類型檢查

### 9.3 自訂斜線指令

透過在 `.claude/commands/` 新增 markdown 檔案來建立自訂指令：

```text
.claude/commands/test.md        -> /test
.claude/commands/builder/plugin.md -> /builder plugin
```

**指令檔案結構範例**（`.claude/commands/test.md`）：

```markdown
為以下檔案撰寫單元測試：$ARGUMENTS
遵循我們的測試慣例：
- 使用 Jest
- 包含邊界情況
- 模擬外部依賴
```

使用方式：`/test src/utils/helpers.ts`

### 9.4 記憶管理

使用 `#` 符號新增記憶：

```text
#新功能一律使用 Mobx 元件
```

**記憶類型**：

- **全域使用者記憶**：適用於所有地方的偏好設定
- **本地專案記憶**：專案特定的記憶（會被 git 忽略）
- **CLAUDE.md**：共享的專案文件

## 11. Claude Code 的優勢

### 11.1 技術優勢

1. **專注單一模型**：只需支援 Claude 模型
2. **深度整合**：工具與模型一同最佳化
3. **持續改進**：模型改進直接嘉惠 Claude Code
4. **專家知識**：由最了解模型的團隊打造

### 11.2 實際效能表現

- 處理超過 18,000 行的檔案毫無問題
- 在大型程式庫中具有優秀的模式識別能力
- 理解元件、狀態與檔案之間的關係
- 極少卡住或需要人工介入

### 11.3 與其他工具的比較

其他 AI 程式工具：

- 必須支援多種模型（增加複雜度）
- 有額外的團隊和層級
- 無法控制核心 AI 模型
- 必須在 API 成本上加收自己的利潤

## 13. 最佳實踐總結

### 13.1 日常工作檢查清單

1. 開始新任務時使用 `/clear`
2. 考慮使用 `--dangerously-skip-permissions` 以獲得不中斷的工作體驗
3. 將多個相關任務排入佇列
4. 複雜任務使用 Opus，簡單任務使用 Sonnet
5. 設定 CLAUDE.md 以提供專案特定的內容

### 13.2 生產力技巧

- 開啟多個 Claude Code 視窗平行處理不同檔案
- 使用上方向鍵存取先前的工作階段
- 設定 GitHub 整合以獲得自動化 PR 審查
- 為重複性任務建立自訂斜線指令
- 使用訊息佇列批次處理任務，稍後再回來查看

### 13.3 常見錯誤避免

> **警告**：
> - 不要使用 Ctrl + C 停止工作（請用 Escape）
> - 不要保留冗長的對話記錄（請用 /clear）
> - 不要忘記定期檢查佇列中的任務
> - 不要用 Cmd + V 貼上圖片（請用 Ctrl + V）
